# SF5 竞品应对 UTF-8 测试人工结论

测试时间：2026-05-21

## 结论

SF5 竞品应对已完成 24 条 UTF-8 测试。接口错误 0 条，空回复 0 条，没有乱码。

当前流程可用度中等：基础竞品比价、资质质疑、售后保障类问题能回复，但主路由对竞品意图识别不够稳定，强压价边界处理失败，部分话术存在内部表达和未检索就承诺的问题。

## 表现较好的点

- 多数竞品回复没有直接贬低对方机构。
- 比价类能引导客户比较产品、剂量、部位、服务、售后等维度。
- 产品真假、资质证书、售后保障、机构不放心类问题能返回 `image` 类型资料。
- 没有大量转人工，符合“尽可能少转人工”的方向。

## 主要问题

1. 竞品意图识别不稳定  
   多个明确带“别家、另一家、他们”的消息没有进入 `competitor_compare`：  
   - `SF5_002` 被判为 `bargain`  
   - `SF5_005` 被判为 `price_inquiry`  
   - `SF5_006` 被判为 `campaign_inquiry`  
   - `SF5_011`、`SF5_019`、`SF5_020` 被判为 `project_inquiry`  
   只要当前消息核心是拿别家信息做比较，应优先进入 SF5，除非是明确强压价或投诉。

2. 强压价边界处理失败  
   `SF5_014` “你们要是比他们便宜我就来，不然算了”识别为 `bargain`，但回复只是“您好，我是您的专属客服...”。  
   这里应进入人工或至少输出“价格需要按统一配置，无法线上反复议价，我可以帮你核对项目内容差异”。不能用泛用兜底。

3. 普通比价被误判为砍价  
   `SF5_002` “另一家光子才299，你们为什么贵”只是竞品比价，不是强压价。  
   建议区分：  
   - 询问为什么贵、别家便宜：SF5  
   - 要最低价、必须比别人低：HUMAN_HANDOFF 或价格边界回复

4. 暴露内部系统表达  
   `SF5_005` 出现“我先按系统配置给你说哈”。  
   `SF5_006` 出现“需要为您查询一下系统里的最新信息”。  
   客户侧禁止出现“系统配置/查询系统/知识库”等内部词，应改成“我先按当前可确认的信息说”或“具体活动要以当前活动为准”。

5. 未检索或资料不足时有编造风险  
   `SF5_011` 客户问进口设备，回复“我们这边有进口设备的抗衰项目方向”，但没有返回设备资料图片或明确依据。  
   设备来源、产品溯源、医师资质、售后保障都必须有知识库结果才可具体说。

6. 资质/背书话术承诺过强  
   `SF5_010` 说“所有诊疗项目都在资质范围内”。  
   更稳妥表达应是“可先查看机构资质，到店也可以核对项目和操作范围”，不要线上做绝对覆盖判断。

7. 活动/价格场景和竞品场景边界不清  
   `SF5_006` 竞品活动“买一送一”被活动流程接走，还主动说有新客礼包、老客复购优惠。  
   如果活动系统没有返回明确活动，不应说具体活动类型，只能说“活动以当前配置为准，我可以先帮你看适用项目”。

## 建议调整

1. 主路由增加竞品强识别规则  
   当前消息包含以下词时优先 `competitor_compare -> SF5`：  
   “别家、另一家、他们家、其他机构、同行、报价单、截图、比你们便宜、和你们比、他们说、朋友在别家做”。

2. 竞品与砍价分离  
   - `别家便宜/为什么贵/报价差异`：SF5  
   - `最低价/必须便宜/不便宜就不来/给底价`：HUMAN_HANDOFF 或价格边界回复

3. SF5 提示词增加禁止词  
   禁止输出：系统配置、查询系统、知识库、检索结果、资料库、工具返回。

4. 知识库使用更严格  
   - 说资质：必须有资质背书图片或资质文本  
   - 说产品真假：必须有产品溯源资料  
   - 说设备进口：必须有设备资料  
   - 说售后保障：必须有售后流程资料  
   无资料时只能说“具体资料我帮你进一步确认”，不能编造。

5. 回复结构收紧  
   推荐 2-3 条：  
   - 第 1 条：认可客户对比  
   - 第 2 条：给对比维度或资料  
   - 第 3 条：低压力下一步，只问 1 个问题  

## 可接受分流

部分“别家说你们不正规/产品不正规/售后保障”被路由到 `trust_issue` 并触发 SF10 资源，这在最终业务上可以接受，因为客户核心顾虑是信任。但主路由 reason 和最终报告中应能解释这是“竞品触发的信任问题”，避免误以为丢失竞品场景。
