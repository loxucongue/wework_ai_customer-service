# 后续优化路线图

本文件列“以后可以优化”的内容，优先级低于 `IMMEDIATE_FIX_PLAN.md`。这些优化主要用于提升准确率、自然度、转化效率和长期维护能力。

## 阶段 1：外部数据和配置系统补齐

### 1. 报价系统

用途：服务 `SF7_price_consult`、部分 `SF8_campaign_push`、`SF9_appointment`。

建议先实现简单字段：

- `category_name`：大品类
- `item_name`：项目/产品名称
- `daily_price`：日常单次价
- `new_price`：新客体验价
- `old_price`：老客单次价
- `old_card`：老客推荐卡项
- `active_price`：活动价
- `active_group`：活动适用人群
- `start_time`：活动开始时间
- `end_time`：活动结束时间
- `gift`：可赠送福利
- `gift_scene`：福利触发场景
- `status`：启用/停用
- `note`：内部备注

### 2. 活动系统

用途：服务 `SF8_campaign_push`，避免模型编造活动。

建议活动必须有：

- 活动名称
- 项目/大品类
- 适用人群
- 适用门店
- 生效时间
- 失效时间
- 活动权益
- 活动素材 URL
- 是否启用

### 3. 门店信息库

用途：服务 `SF6_store_match`、`SF9_appointment`。

建议字段：

- 城市
- 区域/商圈
- 门店名称
- 详细地址
- 营业时间
- 交通方式
- 停车信息
- 地图链接
- 支持项目/服务
- 门店状态

### 4. 预约系统接口

用途：服务 `SF9_appointment`。

需要支持：

- 查询可约时段
- 创建预约
- 改约
- 取消预约
- 查询预约状态
- 查询预约门店

### 5. 客户订单/项目记录接口

用途：服务 `SF12_after_sales`、`SF7_price_consult`、`SF8_campaign_push`、`SF9_appointment`。

需要支持：

- 查询客户做过的项目
- 操作时间
- 操作门店
- 疗程进度
- 负责顾问/护理老师
- 项目大类新老客判断

## 阶段 2：知识库质量提升

### 1. 项目知识库

补齐方向：

- 光子嫩肤原理、适合人群、恢复期、风险
- 皮秒/超皮秒淡斑边界
- 水光、冻干粉、PDRN、修护类项目
- 小气泡、清洁类项目
- 热玛吉、超声炮、射频、抗衰提升
- 黑眼圈、毛孔、暗沉、泛红、痘印等需求到项目方向的映射
- 项目对比卡片

### 2. 竞品话术知识库

补齐方向：

- 别家便宜
- 别家买一送一
- 别家说设备更好
- 别家说产品进口
- 别家说你们不正规
- 客户发竞品报价截图
- 客户激烈比价

原则：不贬低竞品，只拉回对比维度和自身可证明资料。

### 3. 售后话术知识库

补齐方向：

- 洗脸
- 化妆
- 运动
- 喝酒
- 防晒
- 敷面膜
- 修复产品
- 红、肿、热、痒、痛
- 结痂、反黑、色沉
- 效果观察周期

### 4. 品牌资质/背书图片库

补齐方向：

- 机构资质
- 产品授权/溯源
- 设备授权/检测
- 医师/技师资质
- 售后流程
- 消毒/安全流程
- 门店环境/规模

每张图片需要标注：

- 适用顾虑类型
- 图片描述
- 可对客解释话术
- 禁用夸大说法
- 图片 URL

### 5. 效果对比图片库

后续可用于信任建立和项目咨询，但必须注意授权。

建议字段：

- 项目名称
- 改善方向
- 客户基础情况摘要
- 时间周期
- 是否授权
- 隐私处理状态
- 图片 URL
- 禁用表达

## 阶段 3：主路由增强

### 1. 当前消息优先

继续强化：当前消息必须压过历史对话。历史只用于补充指代，不用于覆盖当前意图。

### 2. 指代解析

优化“这个多少钱、刚刚那个、你说的项目、它安全吗”等表达，从最近一次明确项目名中提取 `project_namelist`。

### 3. 短确认解析

“可以、收到、好的、嗯嗯”需要结合上一轮助手动作：

- 转人工确认
- 预约确认
- 价格说明
- 活动说明
- 普通说明

### 4. 多意图排序

建议排序：

1. 安全/投诉/人工
2. 预约变更/取消
3. 当前明确业务意图
4. 信任/竞品/价格
5. 画像补充
6. 情绪陪伴/闲聊

## 阶段 4：回复风格统一

### 统一要求

- 默认 1-3 条消息。
- 每条 20-70 个中文字符。
- 一条只讲一个意思。
- 每轮最多问 1 个关键问题。
- 不使用 emoji。
- 少用“呀、哈、哦、好哒、～”。
- 不使用“作为 AI、系统、知识库、工具”等内部表达。

### 推荐结构

项目咨询：

1. 接住问题
2. 给方向和边界
3. 问一个关键问题或引导面诊

信任建立：

1. 接住顾虑
2. 透明流程/资料
3. 低压力下一步

售后：

1. 不在线诊断
2. 收集关键信息
3. 护理老师/门店跟进或保守护理建议

## 阶段 6：灰度上线策略

### 建议顺序

1. 内部测试环境全量回归。
2. 人工审核模式，AI 只给建议回复。
3. 低风险场景自动回复，例如打招呼、轻项目咨询、普通信任问题。
4. 外部接口稳定后开放价格、活动、门店、预约。
5. 售后和高风险场景长期保留人工优先。

### 低风险可先自动

- 新客打招呼
- 简单项目方向咨询
- 普通信任问题
- 轻情绪承接
- 常规术前问题

### 长期人工优先

- 投诉退款
- 效果纠纷
- 严重术后异常
- 孕期/未成年/慢病
- 强压价
- 自伤风险
- 病例/诊断/异常伤口图片
