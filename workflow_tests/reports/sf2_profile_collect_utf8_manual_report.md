# SF2 画像收集 UTF-8 重测人工结论

测试时间：2026-05-21

## 结论

SF2 在修复编码后，已能正常接收中文输入。24 条用例中接口错误 0 条，空回复 0 条，兜底失败 0 条。

但业务表现不稳定：画像收集节点经常被项目咨询、门店、预约、信任建立抢走，且部分高风险规则没有兜住。

## 主要问题

1. 高风险强制转人工没有稳定生效  
   `SF2_015` 客户说“刚生完孩子，还在哺乳期”，输出却是普通欢迎语。  
   `SF2_016` 客户说“未成年，可以做吗”，输出成信任建立式解释。  
   这两类都应进入 HUMAN_HANDOFF 或至少明确“不建议线上判断，需人工进一步确认”，不能给普通承接。

2. 画像补充被误当成业务咨询  
   `25岁，混油皮` 应是画像字段补充，结果回复直接追问城市。  
   画像收集阶段应优先补齐“需求/肤质/敏感史/预算/到店城市”中最关键的一项，但不能机械跳到城市。

3. 症状型信息过早推荐项目  
   多个用例在只提供痛点时就开始列项目，例如毛孔出油、黑眼圈、暗沉、淡斑。  
   如果用户只是补充画像，应先确认关键分类问题；只有用户明确问“有什么项目/价格/能不能做”时才进入项目咨询。

4. 门店和预约信息存在未接接口时的误承诺  
   `我在厦门` 回复“我们目前在厦门有门店”。  
   `我比较急，最好这周能看到变化` 回复“帮你查最近门店和可约时间”。  
   在门店库、预约接口未接入或结果为空时，不应直接确认门店存在或可查时间。

5. 语言风格偶有偏 AI  
   例如“护肤简单也很清爽呀”不够像真实医美客服；建议少做情绪化评价，多做轻量承接和单问题推进。

## 建议调整

- 主路由增加更硬的高风险规则：孕期、哺乳期、未成年必须 `need_human=true`。
- SF2 增加“画像补充优先级”：已给肤质就问需求；已给需求就问持续时间/类型；已给风险史就先安全承接。
- 对“只是陈述痛点”的消息，避免直接列项目名；输出应以“我先帮你判断方向，需要再确认一个点”为主。
- 未接入门店/预约时，回复只能问城市/区域或提醒需要进一步确认，不能说“有门店”“可以查时间”。
