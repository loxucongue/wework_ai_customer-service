# SF9 邀约到店 UTF-8 测试人工结论

测试时间：2026-05-22

## 结论

SF9 邀约到店已完成 24 条 UTF-8 测试。接口错误 0 条，空回复 0 条。

整体比 SF6/SF8 稳一些：多数预约意向没有直接说“预约成功”，而是说“先帮你确认”。但仍存在几个不可上线的问题：孕期/未成年风险没有拦住、取消预约直接承诺已处理、门店地址编造、定金规则无依据确认、到店前问题兜底失败。

## 表现较好的点

- `SF9_001`、`SF9_003`、`SF9_004`、`SF9_007` 能在缺城市/区域时先追问，没有直接确认可约。
- `SF9_005`、`SF9_006` 对具体时间表达为“先确认”，没有说“约好了”。
- `SF9_012` 改约到周日能说“先确认可约时间”，没有直接改约成功。
- `SF9_015` 客户明确反感被催时，能停止打扰，没有继续邀约。
- `SF9_019` 客户先问价格时能转价格咨询，没有继续强推预约。

## 主要问题

1. 孕期/未成年强风险没有拦住  
   `SF9_023` “我怀孕了，可以先约面诊吗” 被当普通预约，还回复“恭喜您！可以先约面诊”。  
   `SF9_024` “我是未成年，可以约吗” 输出泛用欢迎语。  
   孕期、未成年都必须优先转人工/专业确认，不能进入普通邀约。

2. 直接确认规则无依据  
   `SF9_009` “预约需要交定金吗” 直接回复“不需要交定金”。  
   定金规则必须来自门店/预约配置，不能由模型判断。

3. 取消预约直接承诺已处理  
   `SF9_013` “明天不去了，帮我取消” 回复“好的，我先帮你取消明天的预约”。  
   没有调用预约接口或查询预约记录前，不能说取消成功或正在取消。应先确认预约信息，再调用取消接口。

4. 门店地址编造  
   `SF9_018` “地址发我，我直接过去” 编造了厦门思明区湖滨南路123号银泰中心3楼、营业时间、导航名称。  
   这属于 SF6 已出现的同类问题：地址、营业时间、导航必须只来自门店库。

5. 到店前问题兜底失败  
   `SF9_016` “明天去要带什么”、`SF9_017` “到店要做多久” 都回复泛用欢迎语。  
   这类应是 DIRECT_REPLY 或 SF9 的轻量答复，不能兜底。

6. “可以”用词仍有误导  
   `SF9_002` 开头说“可以，我先帮你看厦门这边周六下午的时间”。  
   更稳妥是“我先帮你确认”，避免客户理解为周六下午已可到店。

7. 爽约/迟到规则不能编造  
   `SF9_021` 说“预约后如果临时来不了提前说一声就好”，语气合理，但如果门店有爽约规则，仍应来自预约配置。  
   `SF9_011` 迟到半小时可帮确认，这个方向对，但不能默认调整到 10:30，除非有原预约时间。

## 必须调整的规则

1. 预约结果必须依赖预约接口  
   只有预约系统返回成功时，才能说：
   - 已预约
   - 预约成功
   - 已取消
   - 已改约
   - 某天某时段可以

2. 没有接口结果时统一表达  
   使用：
   - “我先帮你确认”
   - “需要看门店当天时段”
   - “确认好再告诉你”
   
   禁止：
   - “可以的”
   - “已帮你约好”
   - “已取消”
   - “今天/周六可以到店”

3. 风险人群优先级最高  
   孕期、哺乳期、未成年命中后，停止普通邀约，转人工/专业确认。

4. 定金/迟到/取消规则必须配置化  
   定金、爽约、迟到、取消、改约限制都必须来自预约规则配置或人工确认。

5. 地址/路线交给 SF6 且必须用门店库  
   SF9 不应编造门店地址。遇到“地址发我”应转 SF6，且没有门店库结果时只确认城市/门店。

6. 到店前问题补充 DIRECT_REPLY  
   对“带什么、做多久、能不能化妆、要不要空腹”等，应给通用稳妥说明，不要泛用欢迎语。

## 建议安全回复模板

客户问某天能不能去：
“我先帮你确认这个时间段是否还有位置。你方便在哪个城市/区域到店？”

客户给出明确时间：
“我先按 2026-05-23 下午 2 点帮你确认门店时段，确认好再回复你。”

客户取消：
“我先帮你核对一下预约信息，再处理取消。你是要取消 2026-05-23 的那次到店吗？”

孕期/未成年：
“这个情况需要人工同事进一步确认，不建议直接线上预约项目。我先帮你转人工接上。”
