# SF3 项目咨询 UTF-8 重测人工结论

测试时间：2026-05-21

## 结论

SF3 项目咨询在 UTF-8 修复后整体可运行。24 条用例中接口错误 0 条，空回复 0 条，23 条识别为 `project_inquiry`，1 条识别为 `trust_issue`。

整体方向可用，但存在知识库覆盖不足、回复边界不稳、个别话术暴露内部机制的问题。当前不建议直接上线为最终版本，需要先调整知识库和 SF3 提示词。

## 表现较好的点

- 大多数项目咨询能给出“方向 + 追问 + 面诊边界”。
- 对敏感肌、水杨酸、反黑、恢复期等问题没有做绝对承诺。
- 多项目对比类问题基本能拆开讲，例如皮秒/光子、热玛吉/超声炮。
- 没有出现接口失败、空回复或乱码。

## 主要问题

1. 项目知识库覆盖或检索不足  
   `SF3_004` 问“光子嫩肤是什么原理”，回复“暂时无法详细说明”。  
   `SF3_020` 问“PDRN三文鱼冻干修护套组主要修复什么”，回复“暂时无法提供”。  
   这两个都属于项目知识库应覆盖的基础问题。

2. 暴露内部知识库机制  
   `SF3_023` 回复“知识库中暂未获取到小气泡相关信息”。  
   客户侧不能看到“知识库”这类内部词，应改成“这个项目我先不展开说，具体以到店确认资料为准”。

3. 个别回复承诺感偏强  
   `SF3_002` 说“可以改善毛孔粗大和出油”，建议改为“有对应改善方向”。  
   项目咨询节点应避免直接保证改善，只能说方向、适配条件和面诊确认。

4. 安全/信任问题容易跨到 SF10  
   `SF3_018` “做淡斑会不会伤皮肤”被识别为 `trust_issue`，并发送了资质/流程图片。  
   这个方向可以接受，但话术“严格遵循无菌流程”“能有效降低感染风险”偏硬，建议改为更保守表达：会先评估皮肤状态、说明风险和护理要求，不适合不硬做。

5. 项目推荐仍有过早问题  
   `SF3_021` 用户只说“暗沉和毛孔”，回复直接给“光子嫩肤和水杨酸焕肤”。  
   如果没有充分画像，建议先问 1 个关键区分问题，例如“出油闭口明显，还是单纯肤色暗沉更明显？”

6. 场景识别偶尔受历史影响  
   `SF3_012`、`SF3_020` 进入 `S7_dealed_active`，如果客户只是普通咨询维持时间/产品功效，应更偏 S3；只有明确做过项目或售后问题才按 S7/SF12。

## 建议调整

- 项目知识库补齐基础条目：光子嫩肤原理、PDRN三文鱼修护、水光/小气泡对比、淡斑风险边界。
- SF3 提示词增加硬规则：禁止输出“知识库、检索、资料库、暂未获取”等内部词。
- 将“可以改善/能改善”统一改为“有改善方向/可作为参考方向/可能适合某类情况”。
- 对需求未细分的项目推荐，先追问 1 个关键问题，再给项目方向。
- 对安全顾虑类项目问题，允许转 SF10，但回复必须保守，不要把背书图片当作安全证明。

## 可继续测试重点

- SF4 面诊图片咨询：重点看是否能专业描述皮肤状态、诱因和边界。
- SF5 竞品应对：重点看是否主动查竞品话术库和背书图库，且不贬低竞品。
