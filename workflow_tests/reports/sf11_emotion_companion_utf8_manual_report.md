# SF11 情感陪伴 UTF-8 测试人工结论

测试时间：2026-05-22

## 结论

SF11 情感陪伴已完成 24 条 UTF-8 测试。接口错误 0 条，空回复 0 条。

整体可用：轻情绪大多能被自然承接，且项目、价格、售后异常、投诉等没有全部被情感陪伴吞掉。但仍存在投诉兜底失败、部分情绪场景被项目/价格节点抢走后不够有人味、以及信任类回复承诺感偏强的问题。

## 表现较好的点

- 轻情绪如“焦虑、随便看看、自卑、不想花钱、看不懂”等，大多能先接住，再低压力追问。
- “算了不想聊了”“先冷静一下”“别回了”能停止推进，没有继续强邀约。
- 项目选择问题“水光和光子怎么选”能进入项目咨询。
- 术后异常“红肿疼”能进入售后，不被情感陪伴吞掉。
- AI/机器人调侃能坦诚是智能助手，没有假装真人。

## 主要问题

1. 投诉强风险仍然兜底失败  
   `SF11_016` “你们别骗我，我要投诉” 被识别为 `complaint_refund`，但回复泛用欢迎语。  
   这和 SF10 同类问题一致：投诉/维权必须转人工，不能输出欢迎语。

2. “你们老催我到店”处理失败  
   `SF11_008` 被识别为 `complaint_refund`，但同样回复泛用欢迎语。  
   这类不一定要强转人工，但至少要道歉降压、停止邀约，不能欢迎语兜底。

3. 情绪 + 项目痛点时过早推荐项目  
   `SF11_005` “斑真的很烦” 直接进项目咨询并推荐光子/皮秒。  
   更好是先承接情绪，再问斑的形态或持续时间，不要马上报项目。

4. 价格焦虑缺少情绪承接  
   `SF11_013` “太贵了，负担不起” 进入价格咨询，但回复里出现“系统统一口径”，且没有充分承接预算压力。  
   价格节点也需要加情绪承接和禁止内部词。

5. 反复纠结被价格节点抢走  
   `SF11_019` “问了很多次还是不确定” 进入 `price_inquiry`，只问水光还是光子价格。  
   这种应先总结客户当前纠结点，再给一个低压力下一步。

6. 信任类回复承诺感偏强  
   `SF11_014` “怕做坏” 回复“全程有专人跟进、你可以放心”。  
   `SF11_020` “恢复期影响上班” 发术后护理图，但没有回答恢复期和上班安排，且偏资料堆砌。  
   应改成“会先评估、说明恢复期，不适合不建议做”，避免“放心”式承诺。

7. 语言略微偏闲聊/emoji  
   `SF11_006` 使用 emoji，`SF11_017` “陪你聊～最近有没有想分享的事情”偏泛闲聊。  
   医美企微客服可以温和，但应快速回到客户需求，避免变成陪聊机器人。

## 必须调整的规则

1. 投诉/退款/维权优先级最高  
   命中 `complaint_refund` 后，不进入 SF11，必须转人工或输出转人工承接。

2. 短负面情绪处理  
   对“算了、不想聊了、别回了、先冷静、烦你们催”：
   - 停止业务推进
   - 简短道歉或尊重暂停
   - 不追问项目/预约

3. 情绪 + 业务问题的回复顺序  
   应先情绪承接，再处理业务：
   - “听起来你是真的被这个问题困扰了”
   - “我先不急着推项目，先帮你把情况理清楚”
   - 每轮最多问 1 个关键问题

4. 禁止内部词  
   SF11 调到价格或活动相关回复时，也要继承禁止词：系统价、系统配置、系统统一口径。

5. 不做心理陪聊泛化  
   SF11 不是长期陪聊节点。回复应控制在 1-2 条，尽量回到需求、顾虑或暂停尊重。

6. 不使用 emoji  
   企微客服风格建议稳定、自然即可，不使用 emoji，避免不专业。

## 建议安全模板

焦虑纠结：
“纠结很正常，先不用急着决定。我先帮你把最担心的点理清楚，你现在更担心效果、恢复期，还是价格？”

不想聊：
“好的，我先不打扰你。后面你想继续了解的时候再找我就行。”

自卑：
“先别这么否定自己，很多皮肤和轮廓问题都可以先按方向了解。你现在最在意的是哪一块？”

被催烦了：
“抱歉让你有压力了，我这边先不继续邀约。你想了解的时候再慢慢问就行。”

投诉：
“我理解你现在比较不满，这个我帮你转人工同事处理。”
