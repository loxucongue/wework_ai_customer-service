# SF1 新客首响 UTF-8 重测人工结论

测试时间：2026-05-21

## 结论

SF1 在修复请求编码后，已不再出现中文乱码导致的“看不清/没获取到内容”类异常。

24 条用例中，接口错误 0 条，空回复 0 条，明显兜底失败 0 条，转人工 0 条。

## 表现较好的点

- 纯打招呼类消息可以稳定进入 S1，并给出首响问题。
- 客户带明确需求时，系统会按当前消息进入项目咨询、活动咨询或情绪承接，没有再被乱码干扰。
- 回复整体较短，没有出现大段百科式内容。

## 仍需优化

1. 新客首响和项目咨询边界需要更明确  
   例如“你们这边能改善皮肤吗”“我想改善毛孔和出油”会直接进入项目咨询，并开始提具体项目方向。对新客第二句话以前，建议先问需求维度，少报项目名。

2. 不应在资料不足时提具体项目  
   `SF1_023` 回复中提到“水杨酸焕肤、微针”，这对新客首响来说偏早。建议主路由或 SF3 增加规则：客户只描述症状、没有追问“有什么项目”时，先收集 1 个关键字段，不直接列项目。

3. “第一次做想了解流程”回复过于兜底  
   `SF1_019` 输出“有什么可以帮您的吗？”不承接“第一次做”和“了解流程”。建议补充 DIRECT_REPLY 或 SF1 规则：先说明“可以先了解，不急着决定”，再问关注方向。

4. 活动咨询在项目未知时要更保守  
   `SF1_021` 提到新客专享体验活动和指定门店，但当前没有活动系统结果。建议活动子流程没有 `campaign_info` 时，不要说具体活动，只能说“不同项目活动不一样，我先帮你看适合方向”。
