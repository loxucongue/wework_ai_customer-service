# SF4 面诊图片咨询 UTF-8 重测人工结论

测试时间：2026-05-21

## 结论

SF4 面诊图片咨询已完成 24 条 UTF-8 重测。接口错误 0 条，空回复 0 条，没有乱码。

但当前 SF4 不建议直接上线。主要问题不是“不会回复”，而是看图类意图识别不够稳定、图片输入使用不一致、回复过长，以及部分场景过早进入项目推荐。

## 数据概览

- 总用例：24
- 接口错误：0
- 空回复：0
- `image_inquiry`：13
- `project_inquiry`：8
- `emotion_chat`：3
- 单条过长：11 条

## 表现较好的点

- 大部分看图问题没有做医学诊断，能提示线上判断有限、需面诊确认。
- 对“是不是敏感肌”“是不是过敏”“有没有炎症”这类风险表达整体较谨慎。
- 对暗沉、毛孔、泛红、痘印、下颌线等问题能给出一定观察方向。
- 没有明显主动转人工过多，符合“尽可能少转人工”的目标。

## 主要问题

1. 看图意图识别不够硬  
   `SF4_004` “看下我的斑属于哪种”、`SF4_007` “帮我看看黑眼圈是什么类型”、`SF4_008` “帮我看下面部松弛严不严重”等被识别成 `project_inquiry`。  
   只要当前消息包含“照片、图、看、帮我看、上传、拍照、侧脸、素颜照”等，或 `file_image` 非空，应优先进入 `image_inquiry`/SF4。

2. 图片输入使用不一致  
   多个用例带了 `file_image`，但回复表现像没看到图片：  
   - `SF4_002` 说“你的文字我收到啦”  
   - `SF4_006` 说“照片如果没上传的话”  
   - `SF4_020` 说“你可以先提供照片”  
   这说明图片变量可能没有稳定进入视觉理解，或 SF4 提示词没有强制区分 `file_image` 是否为空。

3. 回复过长，不适合企微  
   11 条出现单条过长，典型如 `SF4_001`、`SF4_003`、`SF4_010`、`SF4_017`。  
   需要固定拆成 2-3 条：第一条观察，第二条可能因素/边界，第三条只问 1 个问题。

4. 过早推荐项目  
   `SF4_014` 法令纹和苹果肌直接提玻尿酸填充、胶原刺激。  
   `SF4_020` 发照片直接推荐光子和水光。  
   面诊图片咨询应先做观察和需求确认，除非客户明确要求“推荐项目”，也只能给方向，不能定方案。

5. 模糊/光线问题处理偏情绪化  
   `SF4_013` 照片糊，回复“你的样子本身是很有特点”，没有解决拍照质量问题。  
   这类应明确说明：模糊会影响判断，可先看大方向，建议自然光、正脸、侧脸、无滤镜补拍。

6. 非本人照片缺少边界  
   `SF4_024` 非本人照片只做泛泛承接。  
   应补充：只能做一般观察，不能代替本人方案判断，建议本人授权/本人照片或面诊确认。

## 建议调整

1. 主路由加硬规则  
   `file_image` 非空时，除投诉/严重售后/明确人工外，优先 `image_inquiry -> SF4_face_consult`。

2. SF4 输入规则加硬  
   - 如果 `file_image` 非空：必须基于图片做观察，但要说明光线/角度限制。  
   - 如果 `file_image` 为空：禁止说“照片我看到了”，改为拍摄指引或口述引导。

3. 固定回复模板  
   - 第 1 条：我先按照片能看到的范围说。  
   - 第 2 条：观察到的皮肤/轮廓状态 + 可能影响因素。  
   - 第 3 条：边界 + 一个关键追问。  
   每条 20-70 字，避免单条长段。

4. 图片分析维度标准化  
   皮肤类按：肤色均匀度、色沉/斑点、毛孔/纹理、泛红/屏障、干燥油脂感。  
   轮廓类按：下颌线、苹果肌、法令纹、中下面部松弛、脂肪/凹陷倾向。  
   只能说“看起来/可能/倾向”，不能诊断。

5. 项目推荐收紧  
   SF4 中默认不直接定项目。客户明确问“适合什么项目”时，最多给 1-2 个方向，并加“需面诊确认”。
